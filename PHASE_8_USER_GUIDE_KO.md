## 개요

Phase 8은 **청구 및 정산 자동화 시스템**으로, 청구서 생성, 결제 수금, 기사 정산 프로세스를 자동화하여 수작업을 96% 줄이고 현금 흐름 관리를 개선합니다.

### 주요 이점
- ⏱️ **시간 절감**: 청구서 처리 시간 2시간 → 5분
- 🚀 **실시간 정산**: 정산 검토 3일 → 실시간
- 💰 **수금 개선**: 정시 결제율 85% → 100%
- ✅ **오류 감소**: 오류율 3-5% → 0.1% 미만

### 접속 정보
- **URL**: http://139.150.11.99/
- **관리자 계정**: admin / admin123

## 1. 재무 대시보드

### 📊 화면 설명
재무 대시보드는 회사의 재무 현황을 한눈에 볼 수 있는 메인 화면입니다.

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"재무 대시보드"** 선택

### 주요 정보

#### 💵 수익 지표
- **총 수익**: 기간 내 총 매출액
- **청구 금액**: 발행된 청구서 총액
- **수금 금액**: 실제 입금된 금액
- **수금률**: 청구 대비 수금 비율

#### 📈 미수금 현황
- **총 미수금**: 아직 받지 못한 금액
- **정상 미수금**: 만기 전 미수금
- **연체 미수금**: 만기 후 미수금
- **연체 건수**: 연체된 청구서 수

#### 🔄 현금 흐름
- **현금 유입**: 입금된 금액
- **현금 유출**: 지출된 금액
- **순 현금 흐름**: 유입 - 유출

### 사용 팁
✅ 매일 아침 대시보드를 확인하여 재무 상태 파악  
✅ 연체 미수금이 증가하면 결제 알림 발송  
✅ 수금률이 낮으면 고객별 결제 이력 검토

## 2. 요금 미리보기

### 🔍 화면 설명
배송 요금을 실시간으로 계산할 수 있는 도구입니다.

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"요금 미리보기"** 선택

### 요금 계산 방법

#### 1단계: 기본 정보 입력
- **거리 (km)**: 배송 거리 입력
  - 예시: 서울 → 부산 = 400km
- **무게 (kg)**: 화물 무게 입력
  - 예시: 500kg
- **팔레트 수**: 팔레트 개수 입력
  - 예시: 2개

#### 2단계: 옵션 선택
- **긴급 배송**: 체크박스 선택 시 추가 요금 부과
- **온도 제어**: 냉동/냉장 배송 시 선택

#### 3단계: 요금 확인
- **계산** 버튼 클릭
- 화면에 총 요금 표시

### 요금 산정 기준
```
기본 요금 = (거리 × 거리 단가) + (무게 × 무게 단가) + (팔레트 × 팔레트 단가)
긴급 배송 = 기본 요금 × 1.3 (30% 추가)
온도 제어 = 기본 요금 × 1.2 (20% 추가)
```

### 사용 예시
**예시 1: 일반 배송**
- 거리: 100km
- 무게: 500kg
- 팔레트: 2개
- 긴급: 아니오
- **예상 요금**: ₩150,000

**예시 2: 긴급 배송**
- 거리: 100km
- 무게: 500kg
- 팔레트: 2개
- 긴급: 예
- **예상 요금**: ₩195,000 (30% 추가)

## 3. 자동 청구 스케줄

### 📅 화면 설명
고객별로 자동 청구 일정을 설정하여 매월 자동으로 청구서를 발행합니다.

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"자동 청구 스케줄"** 선택

### 스케줄 설정 방법

#### 1단계: 고객 선택
1. **"+ 새 스케줄"** 버튼 클릭
2. 드롭다운에서 고객 선택

#### 2단계: 청구 일자 설정
- **청구일**: 매월 몇 일에 청구서를 발행할지 선택
  - 예시: 매월 5일
  - 범위: 1일 ~ 28일

#### 3단계: 이메일 설정
- **자동 이메일 발송**: 청구서 발행 시 자동으로 이메일 발송
  - ✅ 활성화 권장

#### 4단계: 알림 설정
- **결제 알림 발송**: 만기 전/후 자동 알림
  - **알림 일수**: 만기 며칠 전에 알림을 보낼지 설정
  - 권장: 3일 전, 7일 전

#### 5단계: 저장
- **저장** 버튼 클릭

### 스케줄 관리

#### 활성화/비활성화
- 스케줄 목록에서 **활성화** 토글 스위치 클릭
- 비활성화 시 자동 청구 중지

#### 수정
- 스케줄 목록에서 **수정** 버튼 클릭
- 설정 변경 후 저장

#### 삭제
- 스케줄 목록에서 **삭제** 버튼 클릭
- 확인 후 삭제

### 실행 이력 확인
- **마지막 생성일**: 최근 청구서 발행 일자
- **다음 생성일**: 다음 청구서 예정 일자
- **오류 내역**: 발행 실패 시 오류 메시지 표시

### 사용 팁
✅ 고객별 결제일에 맞춰 청구일 설정  
✅ 이메일 발송은 항상 활성화 권장  
✅ 알림은 3일 전과 7일 전 2회 설정 권장  
✅ 매월 1일에 이력 확인하여 오류 점검

## 4. 정산 승인

### 👤 화면 설명
기사 정산 내역을 검토하고 승인/반려하는 관리자 전용 기능입니다.

⚠️ **권한**: 관리자(Admin) 계정만 사용 가능

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"정산 승인"** 선택

### 정산 승인 프로세스

#### 1단계: 대기 중인 정산 확인
- 화면 상단에 **"승인 대기"** 목록 표시
- 각 항목에 기사명, 정산 금액, 요청일 표시

#### 2단계: 상세 내역 검토
1. 정산 항목 클릭하여 상세 내역 확인
2. 배송 건수 및 요금 확인
3. 공제 항목 확인 (보험료, 유류비 등)
4. 최종 정산 금액 확인

#### 3단계: 승인 또는 반려

**승인하기**
1. **"승인"** 버튼 클릭
2. 승인 메모 입력 (선택사항)
3. **확인** 버튼 클릭
4. 정산 금액이 기사 계좌로 입금 예정

**반려하기**
1. **"반려"** 버튼 클릭
2. **반려 사유** 입력 (필수)
3. **확인** 버튼 클릭
4. 기사에게 반려 알림 발송

### 정산 이력 확인
- **승인 완료** 탭: 승인된 정산 목록
- **반려됨** 탭: 반려된 정산 목록
- **전체** 탭: 모든 정산 이력

### 정산 통계
- **이번 달 승인 금액**: 당월 총 승인 금액
- **이번 달 승인 건수**: 당월 총 승인 건수
- **평균 정산 금액**: 건당 평균 금액

### 사용 팁
✅ 매주 월요일 정산 승인 처리 권장  
✅ 반려 시 명확한 사유 입력 필수  
✅ 정산 금액이 평소보다 크게 다를 경우 재검토  
✅ 승인 이력은 월별로 다운로드하여 보관

## 5. 결제 알림

### 📧 화면 설명
고객에게 결제 알림을 자동으로 발송하는 기능입니다.

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"결제 알림"** 선택

### 알림 유형

#### 1. 만기 전 알림 (BEFORE_DUE)
- **발송 시기**: 만기일 N일 전
- **목적**: 결제 준비 유도
- **권장 설정**: 7일 전, 3일 전

#### 2. 만기일 알림 (DUE_DATE)
- **발송 시기**: 만기일 당일
- **목적**: 정시 결제 독려

#### 3. 연체 알림 (OVERDUE)
- **발송 시기**: 만기일 이후 N일마다
- **목적**: 빠른 결제 요청
- **권장 설정**: 1일 후, 3일 후, 7일 후

### 알림 발송 채널

#### 📧 이메일
- **기본 채널**: 항상 활성화
- **내용**: 청구서 요약, 결제 방법, 고객센터 연락처

#### 📱 SMS
- **보조 채널**: 이메일 미확인 시 사용
- **내용**: 간단한 결제 안내 (140자 이내)

#### 🔔 푸시 알림
- **모바일 앱 전용**: 모바일 앱 설치 고객만 가능
- **내용**: 실시간 알림 배지

### 알림 설정 방법

#### 1단계: 알림 생성
1. **"+ 새 알림"** 버튼 클릭
2. 청구서 선택 (드롭다운)

#### 2단계: 알림 유형 선택
- 만기 전 / 만기일 / 연체 중 선택
- 만기 전 선택 시 **일수** 입력 (예: 7일 전)

#### 3단계: 발송 채널 선택
- 이메일: 기본 활성화
- SMS: 선택 가능
- 푸시: 선택 가능

#### 4단계: 발송 예약
- **즉시 발송**: 지금 바로 발송
- **예약 발송**: 특정 날짜/시간 지정

#### 5단계: 저장
- **저장** 버튼 클릭

### 알림 이력 확인

#### 발송 완료
- 상태: **SENT** (녹색)
- 발송일시 표시

#### 발송 실패
- 상태: **FAILED** (빨간색)
- 오류 메시지 표시
- **재발송** 버튼으로 다시 시도

#### 발송 대기
- 상태: **PENDING** (회색)
- 예약 발송 대기 중

### 알림 성과 분석
- **발송 성공률**: 전체 중 성공 비율
- **이메일 열람률**: 이메일 열람 비율 (향후 추가)
- **SMS 수신률**: SMS 수신 확인 비율

### 사용 팁
✅ 만기 7일 전 + 3일 전 + 당일 3회 발송 권장  
✅ 이메일 + SMS 2개 채널 동시 사용 권장  
✅ 연체 시 매일 알림 발송  
✅ 발송 실패 시 고객 연락처 재확인

## 6. 데이터 내보내기

### 📥 화면 설명
청구서, 정산, 거래 데이터를 Excel 또는 PDF로 다운로드할 수 있습니다.

### 접속 방법
1. 사이드바에서 **"청구/정산"** 클릭
2. **"데이터 내보내기"** 선택

### 내보내기 유형

#### 1. 청구서 (INVOICE)
- 청구서 목록 및 상세 내역
- 고객별, 기간별 필터 가능

#### 2. 정산 (SETTLEMENT)
- 기사 정산 이력
- 승인/반려 상태 포함

#### 3. 거래 (TRANSACTION)
- 입출금 거래 내역
- 현금 흐름 분석용

### 내보내기 방법

#### 1단계: 내보내기 작업 생성
1. **"+ 새 내보내기"** 버튼 클릭
2. **데이터 유형** 선택 (청구서/정산/거래)

#### 2단계: 필터 설정
- **기간 설정**: 시작일 ~ 종료일
- **고객 선택**: 특정 고객만 포함 (선택사항)
- **상태 필터**: 특정 상태만 포함 (선택사항)

#### 3단계: 파일 형식 선택
- **Excel (.xlsx)**: 데이터 분석용
- **PDF (.pdf)**: 인쇄 및 공유용

#### 4단계: 내보내기 실행
- **내보내기** 버튼 클릭
- 작업 목록에 추가됨

### 작업 상태 확인

#### 대기 중 (PENDING)
- 상태: 회색
- 설명: 작업 대기열에 추가됨

#### 처리 중 (PROCESSING)
- 상태: 파란색
- 설명: 데이터 생성 중
- 시간: 1~5분 소요

#### 완료 (COMPLETED)
- 상태: 녹색
- 설명: 파일 생성 완료
- **다운로드** 버튼 활성화

#### 실패 (FAILED)
- 상태: 빨간색
- 설명: 오류 발생
- 오류 메시지 표시

### 파일 다운로드
1. 완료된 작업에서 **다운로드** 버튼 클릭
2. 파일이 브라우저 다운로드 폴더에 저장됨
3. 파일명 형식: `export_청구서_2024-01-01_2024-01-31.xlsx`

### 파일 보관 기간
- 생성된 파일은 **30일간** 보관
- 30일 후 자동 삭제
- 필요 시 로컬에 백업 권장

### 사용 팁
✅ 월말에 전월 데이터 내보내기 권장  
✅ Excel 형식으로 내보내어 추가 분석  
✅ PDF는 고객 공유용으로 사용  
✅ 대용량 데이터는 기간을 나누어 내보내기

## 문제 해결

### 로그인 문제

#### 문제: 로그인이 안 돼요
**해결 방법:**
1. 사용자명과 비밀번호 다시 확인
   - 사용자명: `admin`
   - 비밀번호: `admin123`
2. 대소문자 구분 확인
3. 브라우저 캐시 삭제 (Ctrl + Shift + R)
4. 다른 브라우저로 시도

### 페이지 로딩 문제

#### 문제: 페이지가 안 열려요
**해결 방법:**
