# 브라우저 캐시 완전 삭제 가이드

## 🚨 중요: 배포 후 반드시 브라우저 캐시를 삭제해야 합니다!

### 왜 캐시를 삭제해야 하나요?
- 프론트엔드 빌드 후에도 브라우저가 **이전 JavaScript 파일**을 계속 사용할 수 있습니다
- 특히 `OrderModal.tsx`와 같은 컴포넌트는 캐시에 저장되어 있을 수 있습니다
- 캐시를 삭제하지 않으면 **새로운 코드가 반영되지 않습니다**

---

## 방법 1: 개발자 도구 사용 (추천 ⭐)

### Chrome / Edge
1. **F12** 키를 눌러 개발자 도구 열기
2. **Network** 탭 클릭
3. **Disable cache** 체크박스 선택
4. 새로고침 버튼 **우클릭** → **Empty Cache and Hard Reload** 선택

### 또는 단축키:
```
Windows: Ctrl + Shift + R (또는 Ctrl + F5)
Mac: Cmd + Shift + R
```

---

## 방법 2: 완전한 캐시 삭제

### Chrome / Edge
1. **Ctrl + Shift + Delete** (Windows) 또는 **Cmd + Shift + Delete** (Mac)
2. **시간 범위**: "전체 기간" 선택
3. **항목 선택**:
   - ✅ 쿠키 및 기타 사이트 데이터
   - ✅ 캐시된 이미지 및 파일
4. **데이터 삭제** 버튼 클릭
5. 브라우저 **완전히 종료**
6. 브라우저 **재시작**
7. http://139.150.11.99 접속

---

## 방법 3: 시크릿 모드 사용 (가장 확실 ✨)

### Chrome
```
단축키: Ctrl + Shift + N (Windows/Linux)
단축키: Cmd + Shift + N (Mac)
```

### Edge
```
단축키: Ctrl + Shift + P (Windows/Linux)
단축키: Cmd + Shift + P (Mac)
```

### 절차:
1. **모든 브라우저 창 완전히 닫기**
2. 시크릿 모드로 **새 창 열기**
3. http://139.150.11.99/orders 접속
4. **주문 등록** 버튼 클릭
5. 모달 확인

---

## 방법 4: 특정 사이트 캐시만 삭제

### Chrome / Edge
1. **F12** → **Application** 탭
2. 왼쪽 메뉴에서 **Storage** 섹션
3. **Clear site data** 버튼 클릭
4. 모든 항목 체크
5. **Clear site data** 확인

## 🔍 여전히 문제가 있다면?

### 1. 브라우저 버전 확인

```
Chrome: chrome://version
Edge: edge://version
```
→ 최신 버전으로 업데이트

### 2. 다른 브라우저에서 테스트

- Edge에서 문제가 있다면 → Chrome에서 테스트

```bash

cd /root/uvis
./verify_current_state.sh
```
