## 📘 2. 기능별 상세 매뉴얼

### 2.1 재무 대시보드

#### 목적
회사의 재무 현황을 실시간으로 모니터링

#### 주요 기능
1. **수익 지표 카드**
   - 총 수익: 전체 매출액
   - 청구 금액: 발행된 청구서 합계
   - 수금 금액: 실제 입금액
   - 수금률: (수금액 / 청구액) × 100

2. **미수금 현황**
   - 총 미수금: 아직 받지 못한 금액
   - 정상 미수금: 만기일 이전
   - 연체 미수금: 만기일 이후
   - 연체 건수: 연체된 청구서 개수

3. **현금 흐름**
   - 현금 유입: 들어온 돈
   - 현금 유출: 나간 돈
   - 순 현금 흐름: 유입 - 유출

4. **기간 선택**
   - 오늘
   - 최근 7일
   - 최근 30일
   - 최근 90일
   - 사용자 정의

#### 사용 시나리오

**시나리오 1: 일일 재무 점검**
```
상황: 매일 아침 재무 상태 확인
절차:
1. 재무 대시보드 접속
2. "오늘" 기간 선택
3. 어제 대비 변화 확인
4. 연체 미수금 증가 시 알림 발송 검토
```

**시나리오 2: 월말 정산**
```
상황: 월말 재무 보고서 작성
절차:
1. "최근 30일" 선택
2. 모든 지표 확인
3. 데이터 내보내기로 Excel 다운로드
4. 보고서 작성
```

#### 팁
- 매일 같은 시간에 확인 (예: 오전 9시)
- 수금률이 85% 미만이면 결제 알림 강화
- 연체 미수금은 즉시 대응

### 2.2 요금 미리보기

#### 목적
고객 문의 시 즉시 배송 요금 계산

#### 입력 항목
1. **거리 (km)**: 0~10,000km
2. **무게 (kg)**: 0~50,000kg
3. **팔레트 수**: 0~100개
4. **긴급 배송**: 체크박스 (30% 추가)
5. **온도 제어**: 체크박스 (20% 추가)

#### 요금 계산식
```
기본 요금 = (거리 × ₩1,500) + (무게 × ₩100) + (팔레트 × ₩50,000)

최종 요금 = 기본 요금 × 긴급 배송 할증 × 온도 제어 할증
```

#### 예시

**예시 1: 일반 배송**
```
입력:
- 거리: 200km
- 무게: 1,000kg
- 팔레트: 3개
- 긴급: X
- 온도: X

계산:
기본 요금 = (200 × 1,500) + (1,000 × 100) + (3 × 50,000)
         = 300,000 + 100,000 + 150,000
         = ₩550,000

최종 요금 = ₩550,000
```

**예시 2: 긴급 냉장 배송**
```
입력:
- 거리: 200km
- 무게: 1,000kg
- 팔레트: 3개
- 긴급: O (×1.3)
- 온도: O (×1.2)

계산:
기본 요금 = ₩550,000
긴급 할증 = ₩550,000 × 1.3 = ₩715,000
온도 할증 = ₩715,000 × 1.2 = ₩858,000

최종 요금 = ₩858,000
```

#### 사용 팁
- 고객 문의 즉시 계산
- 견적서 작성 전 확인
- 할인 가능 여부는 별도 협의

### 2.3 자동 청구 스케줄

#### 목적
고객별 자동 청구 일정 설정

#### 설정 방법

**Step 1: 스케줄 생성**
```
1. "+ 새 스케줄" 클릭
2. 고객 선택
3. 청구일 설정 (1~28일)
4. 이메일 자동 발송: 체크
5. 알림 일수: 3일 전, 7일 전
6. 저장
```

**Step 2: 활성화**
```
1. 스케줄 목록에서 활성화 토글
2. 다음 생성일 확인
```

**Step 3: 모니터링**
```
1. 주기적으로 실행 이력 확인
2. 오류 발생 시 고객 정보 업데이트
```

#### 권장 설정

| 고객 유형 | 청구일 | 알림 | 비고 |
|----------|--------|------|------|
| 대기업 | 5일 | 7일 전, 3일 전 | 결제 프로세스 긴 편 |
| 중소기업 | 15일 | 3일 전 | 빠른 결제 가능 |
| 개인 | 말일 | 7일 전, 3일 전, 당일 | 알림 강화 필요 |

### 2.4 정산 승인

#### 목적
기사 정산 내역 검토 및 승인/반려

#### 승인 프로세스

**Step 1: 정산 확인**
```
1. "정산 승인" 메뉴 클릭
2. "승인 대기" 탭 확인
3. 정산 항목 클릭하여 상세 보기
```

**Step 2: 내역 검토**
```
확인 항목:
- 배송 건수
- 배송 요금 합계
- 공제 항목 (보험료, 유류비)
- 최종 정산 금액
- 첨부 서류
```

**Step 3: 승인/반려 결정**
```
승인:
1. "승인" 버튼 클릭
2. 승인 메모 입력 (선택)
3. 확인

반려:
1. "반려" 버튼 클릭
2. 반려 사유 입력 (필수)
3. 확인
```

#### 승인 기준

**승인 조건**
- 배송 건수 정확
- 요금 계산 정확
- 공제 항목 정상
- 첨부 서류 완비

**반려 사유**
- 배송 건수 불일치
- 요금 계산 오류
- 서류 미비
- 기타 문제

### 2.5 결제 알림

#### 목적
고객에게 결제 알림 자동 발송

#### 알림 유형

1. **만기 전 알림** (BEFORE_DUE)
   - 발송 시기: 만기일 N일 전
   - 목적: 결제 준비 안내
   - 권장: 7일 전, 3일 전

2. **만기일 알림** (DUE_DATE)
   - 발송 시기: 만기일 당일
   - 목적: 정시 결제 독려

3. **연체 알림** (OVERDUE)
   - 발송 시기: 만기일 이후
   - 목적: 빠른 결제 요청
   - 권장: 1일 후, 3일 후, 7일 후

#### 발송 채널

1. **이메일** (기본)
   - 청구서 첨부
   - 결제 방법 안내
   - 고객센터 연락처

2. **SMS** (보조)
   - 간단한 안내 (140자)
   - 결제 링크

3. **푸시 알림** (모바일 앱)
   - 실시간 알림
   - 앱 설치 고객만

### 2.6 데이터 내보내기

#### 목적
데이터를 Excel/PDF로 다운로드

#### 내보내기 유형

1. **청구서** (INVOICE)
   - 청구서 목록 및 상세
   - 고객별, 기간별 필터

2. **정산** (SETTLEMENT)
   - 기사 정산 이력
   - 승인/반려 상태

3. **거래** (TRANSACTION)
   - 입출금 거래 내역
   - 현금 흐름 분석

#### 사용 방법

**Step 1: 내보내기 생성**
```
1. "+ 새 내보내기" 클릭
2. 데이터 유형 선택
3. 기간 설정
4. 형식 선택 (Excel/PDF)
5. "내보내기" 클릭
```

**Step 2: 작업 확인**
```
1. 작업 목록에서 상태 확인
2. "처리 중" → "완료" 대기 (1~5분)
```

**Step 3: 다운로드**
```
1. "다운로드" 버튼 클릭
2. 파일 저장
```

## ❓ 4. FAQ (자주 묻는 질문)

### 일반 질문

**Q1: Phase 8은 누가 사용할 수 있나요?**
A: 관리자(Admin) 계정만 사용 가능합니다. 배차원(Dispatcher)은 조회만 가능합니다.

**Q2: 기존 데이터는 어떻게 되나요?**
A: 모든 기존 데이터는 유지됩니다. Phase 8는 새로운 기능을 추가한 것입니다.

**Q3: 모바일에서도 사용할 수 있나요?**
A: 네, 반응형 디자인으로 모바일에서도 사용 가능합니다.

### 재무 대시보드

**Q4: 수금률은 어떻게 계산되나요?**
A: 수금률 = (수금 금액 / 청구 금액) × 100

**Q5: 기간은 어떻게 설정하나요?**
A: 우측 상단의 드롭다운에서 선택하거나 사용자 정의 기간을 입력하세요.

### 자동 청구 스케줄

**Q6: 청구일을 29일 이상으로 설정할 수 있나요?**
A: 아니요, 1~28일만 가능합니다. 모든 월에 존재하는 날짜만 허용됩니다.

**Q7: 스케줄을 일시 정지할 수 있나요?**
A: 네, 활성화 토글을 OFF로 변경하면 됩니다.

**Q8: 청구서 발송에 실패하면 어떻게 되나요?**
A: 오류 메시지가 기록되고, 관리자에게 알림이 갑니다.

### 정산 승인

**Q9: 승인 권한은 누가 가지나요?**
A: 관리자(Admin) 계정만 승인/반려 가능합니다.

**Q10: 승인 후 취소할 수 있나요?**
A: 아니요, 승인 후에는 취소할 수 없습니다. 신중하게 검토하세요.

**Q11: 반려 시 기사에게 알림이 가나요?**
A: 네, 자동으로 이메일과 푸시 알림이 발송됩니다.

### 결제 알림

**Q12: 알림은 언제 발송되나요?**
A: 설정한 시간에 자동 발송됩니다. (예: 만기 7일 전 오전 9시)

**Q13: SMS 비용은 누가 부담하나요?**
A: 회사에서 부담합니다. SMS는 필요시에만 사용하세요.

**Q14: 알림 발송에 실패하면 재발송되나요?**
A: 네, 자동으로 재시도합니다. (최대 3회)

### 데이터 내보내기

**Q15: 내보내기는 얼마나 걸리나요?**

**목표**: 기사 정산 승인 및 지급

**단계**:
1. 정산 승인 메뉴 접속
2. "승인 대기" 탭 확인 (5건)
3. 첫 번째 정산 클릭
4. 내역 검토:
   - 배송 건수: 50건
   - 요금 합계: ₩5,000,000
   - 공제 항목: ₩500,000
   - 최종 금액: ₩4,500,000
5. 정상이면 "승인" 클릭
6. 승인 메모: "정상 처리"
7. 다음 정산으로 이동 (2~5번 반복)
8. 모든 정산 완료 후 "승인 완료" 탭 확인

**확인 사항**:
- 승인 대기 건수가 0인가?
- 승인 완료 건수가 5건인가?
- 각 정산의 금액이 정확한가?
